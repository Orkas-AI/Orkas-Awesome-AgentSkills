# Classroom Implementation

Retrieval practice should feel low-stakes. The goal is learning through recall, not ranking learners.

## Implementation script

Include a short script that covers:

- No notes, textbooks, or devices during the retrieval attempt unless the user asks for an open-note variant.
- Time limit.
- Reassurance that difficulty and partial recall are expected.
- What learners should do if stuck.
- How answers will be reviewed.
- Whether the activity should be collected, self-checked, peer-checked, or discussed.

Example framing:

```text
This is not a test for marks. It is a memory workout. Try first without notes. If you get stuck, write what you can remember, then we will check and fill gaps together.
```

## After the attempt

After learners answer:

- Review correct answers.
- Discuss common errors.
- Ask learners to correct or add to their responses.
- Identify gaps for reteaching.
- Avoid spending all the time on scoring.

## Low-stakes safeguards

Do:

- Use completion, self-check, or discussion.
- Normalize partial recall.
- Emphasize that retrieval effort is part of learning.
- Use results to guide review.

Do not:

- Turn every activity into a grade.
- Publicly rank learners.
- Penalize wrong answers in early retrieval practice.
- Let students use notes during the initial recall attempt if the goal is memory retrieval.

## Variants

Starter:

- Short, fast, no notes.
- Best for core recall and previous lesson review.

Exit ticket:

- Focuses on the day's key ideas.
- Best for checking what needs reteaching.

Spaced review:

- Reuses or rephrases prior questions after days or weeks.
- Best for long-term retention.

Mixed-topic review:

- Combines current and older topics.
- Best after students have enough foundation.

Misconception check:

- Uses flawed statements or scenario contrasts.
- Best when the teacher has observed a repeated error.
