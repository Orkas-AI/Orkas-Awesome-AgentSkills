# Retrieval Task Patterns

Use this reference to schedule active recall tasks inside a study plan. Do not use it to generate a full question bank unless the user explicitly asks for practice materials.

## Retrieval Types

- Free recall: explain, list, write everything remembered, derive from memory, draw from memory. Strong but harder.
- Cued recall: fill blanks, complete a diagram, finish a worked step, use a scenario cue. Good for scaffolding.
- Recognition: multiple choice, matching, choose the correct option. Weaker retrieval effect, useful as warm-up for novices.

## Scheduling Rules

- Use free recall after the learner has seen the material and needs durable retention.
- Use cued recall for early reviews, low-confidence learners, or complex procedures.
- Use recognition only as a short starter or confidence builder.
- Prefer mixed retrieval after multiple topics have been covered.
- Keep retrieval low-stakes. Do not turn every review into a scored exam.

## Task Patterns

```markdown
- Closed-book brain dump: 10 minutes on [topic], then compare with notes.
- Blank diagram: redraw [process/system] and label key parts.
- Error correction: fix 3 deliberate mistakes in a short worked example.
- Teach-back: explain [concept] in plain language in 5 minutes.
- Mixed mini-quiz: 2 questions from each of 3 recent topics.
- Wrong-answer review: redo only the missed items, then write the mistake pattern.
```

## Planner Boundary

The study plan may schedule:

- "Do 8 closed-book recall questions on Topic X."
- "Redo missed questions from Mock Test 2."
- "Write a one-page teach-back for Topic Y."

The study plan should not automatically generate:

- A full worksheet.
- A full exam paper.
- A large item bank.
- Detailed answer keys for many questions.

If the user asks for those materials, treat that as a separate practice-generation task rather than a study-planning step.
