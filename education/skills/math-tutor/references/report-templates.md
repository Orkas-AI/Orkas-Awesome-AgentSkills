# Report Templates

Use reports only when the user asks to summarize learning status, error patterns, or practice records. Use available data only.

## Data Insufficiency

If the user asks for a report but provides little data:

```markdown
I can make a light report from the available problems, but the evidence is limited. For a reliable learning report, I need recent mistakes, practice records, or at least several checked problems.
```

## Boundaries

- Do not invent error statistics.
- Do not claim long-term progress from one or two problems.
- Do not promise score improvement.
- Do not export PDF unless file generation is explicitly requested and the environment supports it.
