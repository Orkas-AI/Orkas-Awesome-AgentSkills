# Planning Workflow

## Required Inputs

Collect these before drafting a plan:

- Measurable goal: the user must be able to tell whether it was achieved.
- Deadline: exact date or remaining days/weeks.
- Current level: score, prior experience, diagnostic result, completed chapters, or self-description.
- Daily time budget: weekday and weekend budgets separately when possible.
- Weak areas: optional in conversation, but ask when the plan would otherwise be generic.

Optional inputs:

- Existing materials, courses, books, tools, or teacher requirements.
- Fixed unavailable days.
- Preferred granularity: monthly, weekly, daily, hourly, or minute-level.
- Output preference: conversation only, Markdown, HTML, printable checklist, or no file.
- Reminder preference and reminder time.

## Plan Shapes

### Short Sprint

Use for 1-14 days.

- Show all days if the plan is short and readable.
- Focus on high-priority targets only.
- Include daily checkpoints and a final review day.
- Avoid adding new resources unless the user already has them.

### Exam Prep

Use for standardized tests, final exams, qualification exams, and subject revision.

- Split into foundation, strengthening, and sprint/consolidation.
- Prioritize high-frequency and high-weight topics first.
- Add spaced review for vocabulary, formulas, concepts, error logs, and weak-area drills.
- Include mock exams only when the user has enough time to review mistakes afterward.

### Long-Term Learning

Use for plans longer than 6 months.

- Start with monthly or weekly milestones, not daily task overload.
- Ask before expanding any month/week into daily detail.
- Keep a review cycle and milestone check every 2-4 weeks.

### Skill Acquisition

Use for programming, design, writing, language production, or other applied skills.

- Include output tasks: build, explain, record, write, present, or teach back.
- Balance input and output; a plan with only videos/readings is incomplete.
- Add small project milestones and demo days.

### Revision Recovery

Use when the user has fallen behind.

- Preserve progress by default.
- Identify missed high-priority tasks, low-priority tasks, and tasks that can be merged.
- Offer options: keep deadline and drop scope, extend deadline, or reduce daily load temporarily.
- Restart with a small task today.

## Progressive Output

For plans longer than 4 weeks:

1. Output phases, weekly/monthly goals, milestones, and workload fit first.
2. Ask whether the user wants weekly detail, all details, or only the first phase.
3. Do not display more than 2 weeks of daily tasks at once unless the user explicitly asks.

For plans of 4 weeks or less:

- It is acceptable to output the full daily schedule when it is not too long.
- Still keep daily tasks concise and checkable.

## Plan Adjustment

When the user says "redo", "regenerate", "start over", or similar:

- Ask whether they want to adjust the current plan while preserving progress or start a new plan.
- Never assume they want to lose prior check-ins, streaks, or completed tasks.

When changing a current plan:

- Summarize what changes.
- Preserve completed tasks.
- Move, merge, or drop uncompleted low-priority tasks.
- Keep a brief version note so the user can understand what changed.
