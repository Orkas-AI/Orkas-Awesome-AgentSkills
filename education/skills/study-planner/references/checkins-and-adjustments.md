# Check-Ins And Adjustments

## Check-In Flow

When the user says they completed tasks, checked in, or finished today's work:

1. Identify today's planned tasks from the current plan context.
2. Ask for completion status only if it is unclear.
3. Report completed, missed, and partially completed tasks.
4. Give brief feedback and tomorrow's next step.
5. If a streak or weekly completion rate is being tracked, update or summarize it only from available data.

## Weekly Report

When the user asks for a weekly report:

- Completion rate.
- Strongest area.
- Weakest area.
- Missed-task pattern.
- Suggested adjustment for next week.
- One short encouraging summary.

Avoid vague praise. Tie feedback to observable plan data.

## Falling Behind

Use recovery-first language:

- "Missing a few days does not mean the plan failed."
- "Let's restart with the smallest useful task."
- "We can preserve the goal by reducing low-priority scope, extending the deadline, or increasing time."

Do not use blame, shame, or pressure.

## Adjustment Options

When the user is behind:

1. Keep the deadline and drop low-priority tasks.
2. Extend the deadline.
3. Temporarily reduce the daily load and focus only on weak/high-priority work.
4. Move missed tasks into a catch-up block if the time budget can absorb them.

When the user adds a new task:

- Place it into the nearest suitable day without breaking the time budget.
- If needed, move or drop lower-priority tasks.

When the user blocks a weekday:

- Shift tasks to nearby days.
- Preserve spaced review timing as much as possible.

## Reminder Support

After the user confirms a plan, ask whether they want reminders unless they already requested them.

If reminders are confirmed:

- Use the user's preferred time if provided.
- Otherwise suggest a default morning reminder.
- Keep reminder content simple: today's tasks, check-in prompt, and streak/status if available.

If the user declines reminders, do not create them.

## Progress Safety

- Never delete an existing plan to "redo" it.
- Never reset check-ins, streaks, or completed tasks without explicit confirmation.
- When the user asks to regenerate, ask whether to adjust the current plan or start a new one.
- If starting a new plan, state that old progress remains separate.
