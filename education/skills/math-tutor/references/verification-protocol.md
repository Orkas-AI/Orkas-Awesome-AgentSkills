# Verification Protocol

Use this before giving math judgments, generated answers, grading feedback, or answer keys.

## Required Steps

1. Read the problem carefully.
2. List known conditions and what is being asked.
3. Solve independently, without relying on the student's answer.
4. Break arithmetic into clear intermediate steps.
5. Verify the final answer:
   - Substitute into the original equation when possible.
   - Check units and reasonableness.
   - Check sign and magnitude.
   - For geometry, check whether lengths, angles, area, and special relations are possible.
   - For probability, check the result is between 0 and 1.
6. Only then state whether the student's answer is correct.

## No Jumping Rule

Do not compress risky arithmetic into one line.

Bad:

```text
9 - 4 + 3 = -2
```

Good:

```text
9 - 4 = 5
5 + 3 = 8
```

## High-Risk Problem Types

Double-check:

- Negative numbers.
- Fractions.
- Roots and radicals.
- Substitution.
- Quadratic formulas.
- Geometry area/length formulas.
- Probability counting.
- Word problems with "increased by" vs "increased to", "area" vs "perimeter", "interest" vs "tax".

## Generated Questions

For each generated problem:

- Solve it independently.
- Confirm the answer is unique when the problem requires uniqueness.
- For multiple choice, verify exactly one option is correct.
- Make distractors wrong for meaningful reasons, not because of ambiguous wording.
- Reject or rewrite any question with conflicting conditions.

## Student Feedback

- If correct, praise after verification.
- If incorrect, name the first key error before showing the full correction.
- If partially correct, preserve the correct idea and repair the wrong step.
