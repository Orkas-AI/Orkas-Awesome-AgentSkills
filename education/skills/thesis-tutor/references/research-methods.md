# Research Methods Guidance

Use this for method design questions. Provide research-method tutoring and feasibility checks; do not fabricate data or perform certification.

## Quantitative research

Common contexts:

- business, education, psychology, sociology, communication, management;
- questionnaire studies;
- variable relationships, hypotheses, and statistical testing.

Five-step workflow:

| Step | Output | Core question |
|---|---|---|
| Research design | variables, hypotheses, model | How is each variable measured? Is each hypothesis theoretically grounded? |
| Pilot test | small sample, reliability/validity check | Does the instrument work before full release? |
| Data collection | valid sample and exclusion rules | How many responses, and what was removed? |
| Data analysis | descriptive, reliability, validity, correlation, regression, mediation, etc. | Which test answers which hypothesis? |
| Result presentation | tables and discussion | What does each table mean for the research question? |

Guidance rules:

- Explain what each analysis is for before naming software steps.
- Ask whether the scale, sample, and hypothesis are already defined.
- Do not invent sample size, questionnaire items, alpha values, p-values, or regression results.
- If the user has output tables, help interpret them cautiously.

## Qualitative research

Common methods:

| Method | Use case | Sample reference |
|---|---|---|
| Semi-structured interview | exploring experience, motivation, or perception | undergraduate 10-15; master's 15-30, if feasible |
| Case study | deep analysis of a phenomenon, organization, or event | 1-4 cases |
| Grounded theory | building a theory from data | usually more demanding; use cautiously for undergraduate work |
| Thematic analysis | identifying recurring patterns across qualitative data | broadly usable |

Thematic analysis frame:

1. Familiarize with the data.
2. Generate initial codes.
3. Search for themes.
4. Review themes.
5. Define and name themes.
6. Write up the analysis with evidence.

Guidance rules:

- Ask for research question, participant source, interview outline, and analysis method.
- Require direct evidence from transcripts or notes for each theme.
- Do not invent interview quotes.
- Do not claim saturation unless the user's data supports it.

## Mixed methods

Ask:

- Which part is primary: quantitative or qualitative?
- Does the qualitative part explain quantitative results, or does it design the quantitative instrument?
- Where do the two data streams connect in the thesis structure?

Common patterns:

- exploratory: qualitative first, then quantitative;
- explanatory: quantitative first, then qualitative follow-up;
- convergent: collect both and compare findings.

## Method limitation language

Help the user state limitations honestly:

- sample size and representativeness;
- data access constraints;
- measurement validity;
- researcher subjectivity;
- context-specific findings;
- correlation vs causation.

Do not turn limitations into excuses; show how the method still supports the research question within scope.
