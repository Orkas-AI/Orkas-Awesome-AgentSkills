# Defense Preparation

Use this when the thesis body is mostly complete and the user is preparing for defense, pre-defense, presentation slides, oral statement, or Q&A practice.

## Defense slide outline

Typical 7-part structure:

| Part | Purpose | Guidance |
|---|---|---|
| Cover | identify project | title, student, program, supervisor, date if required |
| Background | introduce the problem | start from one concrete phenomenon or gap |
| Research question / significance | explain why the work matters | state what existing work misses and what this thesis addresses |
| Method | show credibility | data source, sample/material, method choice, analysis path |
| Results / analysis | present core contribution | use figures/tables where possible; avoid reading dense text |
| Conclusion / limitations | answer the research question | keep to 2-3 main points and state limitations honestly |
| Thanks / Q&A | close | brief and professional |

Do not invent results or visuals. Use only the user's actual thesis content.

## Oral statement

Help the user build a structure, not a fabricated script:

1. Greeting and title.
2. Research background and problem.
3. Research question or objective.
4. Method and data/material.
5. Main findings.
6. Contribution and limitations.
7. Transition to Q&A.

If the user wants a script, provide a fill-in frame with placeholders rather than a finished speech that invents content.

## Common defense questions

| Question type | Example | Answer strategy |
|---|---|---|
| Method challenge | "Why this method?" / "Is the sample size enough?" | explain rationale, feasibility, and limitations |
| Concept clarification | "How do you define X?" | answer from the thesis definition, not improvisation |
| Data source | "Where did the data come from?" | give source, sample, time range, and selection criteria |
| Overclaiming | "How do you prove this effect?" | point to evidence and avoid claims beyond data |
| Innovation | "How is this different from prior work?" | give 1-2 concrete differences |
| Citation | "What does this cited paper argue?" | prepare core points for key references |
| Future work | "What would you do next?" | acknowledge limitations and propose feasible extension |

## Mock defense mode

Workflow:

1. Confirm thesis title, discipline, method, and defense length.
2. Ask one committee-style question at a time.
3. Wait for the user's answer.
4. Give feedback:
   - what worked;
   - what was unclear;
   - what evidence or wording could strengthen it.
5. Continue with the next question.
6. End with a summary of repeated weaknesses and next rehearsal focus.

Do not answer the question for the user before they try.

## Slide critique checklist

- Does every slide have one purpose?
- Are methods and results visible, not hidden in paragraphs?
- Are figures/tables readable?
- Does the conclusion answer the research question?
- Are limitations stated without undermining the whole thesis?
- Is the presentation timed realistically?
