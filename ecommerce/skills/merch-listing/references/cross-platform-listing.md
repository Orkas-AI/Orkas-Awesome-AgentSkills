# Cross Platform Listing

## 标题三层逻辑

1. **搜索层**：类目词、核心功能词、规格词、场景词。
2. **点击层**：痛点、差异、适用人群、场景、价格价值。
3. **合规层**：删除极限词、无证据数据、医疗化表达、侵权品牌词。

## 平台标题差异

| 平台 | 标题重点 | 写法倾向 |
|---|---|---|
| 淘宝 / 天猫 | 搜索词 + 场景 + 人群 | 核心词前置，场景词补充 |
| 拼多多 | 价格价值 + 规格数量 | 简单直接，少抽象品牌话 |
| Amazon | Keyword + Feature + Use Case | 自然嵌入关键词，避免堆砌 |
| Shopify | 品牌感 + SEO | 可读性优先，关键词自然出现 |
| Shopee / AliExpress | 规格 + 兼容性 + 使用场景 | 多语言和单位要准确 |

## 卖点分层

| 层级 | 作用 |
|---|---|
| 功能 | 产品有什么 |
| 利益 | 对用户有什么好处 |
| 证据 | 为什么可信 |
| 场景 | 什么时候用 |
| 反对意见 | 用户担心什么 |

每条卖点优先写成：功能 -> 利益 -> 证据状态。

## FAQ 设计

FAQ 用来处理成交阻力：

1. 规格是否适合我？
2. 怎么安装 / 使用？
3. 是否安全 / 合规？
4. 和普通款有什么区别？
5. 清洁维护是否麻烦？
6. 售后和退换怎么处理？

## 短视频口播草稿

1. 前 3 秒：痛点或场景。
2. 3-8 秒：产品如何解决。
3. 8-15 秒：关键证据或使用画面。
4. 15-25 秒：反对意见处理。
5. 最后：低风险 CTA。
