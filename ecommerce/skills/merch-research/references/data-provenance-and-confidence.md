# Data Provenance And Confidence

## 置信标签

- `直接数据`：来自用户上传、后台导出、授权 API 或可见页面的明确字段。
- `推断`：基于多个字段或材料推理出的判断。
- `方向建议`：策略、预测、进入建议、改良方向。
- `待验证`：缺少来源、样本不足、字段为空、时间不明或口径不稳定。

## Amazon APIClaw 口径

- 缺少 `APICLAW_API_KEY` 时，不执行数据请求。
- 使用接口字段原值，不自行用价格乘销量重算收入。
- 收入优先使用 `sampleAvgMonthlyRevenue`。
- 销量优先使用 `monthlySalesFloor`，并说明它是下限估计。
- 机会指数使用 `sampleOpportunityIndex`。
- 关键词搜索前优先锁定 `categoryPath`，避免跨类目污染。
- 品牌分析必须同时带类目条件。

## 常见误判

- 用单个 ASIN 推断整个类目。
- 用低 BSR 直接等同高利润。
- 忽略评价壁垒、广告成本、FBA 仓储、退货和长期仓储费。
- 用单个平台数据推断全市场。
- 把策略建议标成数据事实。
- 看到高增长就忽略促销、季节性或样本偏差。
